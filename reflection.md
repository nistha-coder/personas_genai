# Reflection (Prompt Engineering Assignment)

Building this persona-based chatbot made me understand that prompt engineering is less about writing fancy sentences and more about designing behavior. At first, my prompts were short and generic, and the responses from the model sounded similarly generic. They were technically correct but not persona-specific. Once I added structured persona descriptions, communication style notes, and constraints, the responses became much more aligned with each person. This directly demonstrated the GIGO principle: if the input prompt is shallow, the output quality is shallow as well.

The most useful change was adding few-shot examples inside each system prompt. With examples, the model had clear guidance on tone, sentence structure, and the type of reasoning expected. For Anshuman Singh, examples with systems language and bottleneck analysis gave a very analytical style. For Abhimanyu Saxena, examples focused on shipping and iteration created practical, execution-oriented responses. For Kshitij Mishra, examples with calm encouragement and fundamentals-first advice made answers mentor-like and patient. This confirmed that examples are one of the strongest tools to shape LLM output.

Another important lesson was constraints. I explicitly added what each persona should avoid, such as emotional hype, aggressive language, or vague theory without action steps. These constraints reduced hallucinated style drift and made the chatbot more consistent across multiple test prompts. The output-format instructions also helped: keeping answers to 4-5 sentences and ending with a question made the conversation feel focused and interactive.

On the product side, the assignment taught me that prompt quality alone is not enough; UX matters too. Persona switching with chat reset, suggestion chips, and a typing indicator made the app easier to use and easier to evaluate. Error handling also mattered because API failures should not crash the interface.

If I continue improving this project, I would add conversation memory per persona, support streaming responses for a more natural feel, and include lightweight analytics to compare response quality across prompts. I would also run a formal prompt evaluation set with fixed questions and score consistency, tone alignment, and actionability. Overall, this project taught me that high-quality AI behavior comes from clear intent, structured context, and iterative prompt refinement.
